# ArchUnit Test Generator

## Purpose

Read the provided architecture inputs and produce a complete, runnable Java ArchUnit
test class that encodes the standards as executable fitness functions.

Rules must be **generic** — written against naming conventions and package patterns,
not specific class names. A rule that references `OrderController` by name only tests
one class. A rule that targets `*Controller` enforces the standard across every
controller on the platform.

---

## Inputs You Will Receive

1. **Service description** — bounded context names, package root, ownership
2. **Architecture standards** — the enterprise rules the platform must meet
3. **ADRs** — specific decisions with their enforcement intent and naming convention notes
4. **API specs** (optional) — inform API contract rules

The source code is not an input. Rules are derived from governance decisions, not from
what the current code happens to look like. If a rule is correct per the ADR, generate
it — the test will pass or fail against the actual code on every build.

---

## How to Generate Rules

### Step 1 — Extract enforceable decisions from each ADR

For each ADR read:
- What is the decision?
- What naming convention does it reference (suffixes, package segments)?
- What is the violation pattern (A must not import B)?

Every rule extracted from an ADR is `ENFORCED CANDIDATE` by default. Mark a rule
`REQUIRES MIGRATION FIRST` only when the ADR itself explicitly notes that a migration
window is needed before enforcement.

### Step 2 — Write generic ArchUnit rules

Each rule must:
- Use `noClasses().that()` with `.haveSimpleNameEndingWith()` or `.resideInAPackage()` — not `.haveFullyQualifiedName()` with a specific class
- Include a `.because()` clause that cites the ADR number and states the business reason
- Be named as a readable sentence describing what the rule enforces, prefixed with the ADR ID for traceability

**Rule naming convention:** `FF_{ADR_NUMBER}_{readable_sentence_in_snake_case}`

Good names read as architecture statements:
- `FF_001_order_context_must_not_import_loyalty_repository`
- `FF_003_controllers_must_only_access_application_layer`
- `FF_017_rest_template_is_banned_use_web_client`
- `FF_SEC_custom_jwt_parsing_must_not_exist_outside_platform_security`

Bad names are opaque identifiers:
- `FF_001_NO_CROSS_CONTEXT_REPO` ✗ — not readable
- `FF_003_LAYER_RULES` ✗ — says nothing about what the rule checks

### Step 3 — Assign status

Each rule gets one of:
- `// Status: ENFORCED CANDIDATE` — default for all rules derived from an ADR
- `// Status: REQUIRES MIGRATION FIRST` — only when the ADR explicitly flags a migration window

### Step 4 — Add migration notes (REQUIRES MIGRATION FIRST only)

```java
// Enable when:
// - [ ] {prerequisite from ADR}
```

## Output Format

Produce a single Java file. Use this exact structure:

```java
package com.example.governance;

import com.tngtech.archunit.junit.AnalyzeClasses;
import com.tngtech.archunit.junit.ArchTest;
import com.tngtech.archunit.lang.ArchRule;
import com.tngtech.archunit.library.dependencies.SliceRule;

import static com.tngtech.archunit.lang.syntax.ArchRuleDefinition.noClasses;
import static com.tngtech.archunit.lang.syntax.ArchRuleDefinition.classes;
import static com.tngtech.archunit.library.dependencies.SlicesRuleDefinition.slices;

/**
 * Platform Architecture Fitness Functions
 *
 * Generated by: ArchUnit Generator skill (AI-assisted)
 * Status: CANDIDATE — architect review required before committing to CI
 *
 * Each rule is tagged with its source ADR and current enforcement status.
 * Rules marked ADVISORY do not currently block builds.
 * Rules marked ENFORCED CANDIDATE are ready to block builds after sign-off.
 * Rules marked REQUIRES MIGRATION FIRST must not be enabled until noted prerequisites complete.
 */
@AnalyzeClasses(
    packages = "{root package from service description}",
    importOptions = com.tngtech.archunit.core.importer.ImportOption.DoNotIncludeTests.class
)
public class GeneratedFitnessFunctionsTest {

    // ── {STANDARD NAME} ──────────────────────────────────────────────────────
    // Source: {ADR number and title}
    // ─────────────────────────────────────────────────────────────────────────

    // Status: {ENFORCED CANDIDATE | ADVISORY | REQUIRES MIGRATION FIRST}
    @ArchTest
    static final ArchRule FF_{ID}_{NAME} =
        noClasses()
            .that()...
            .should()...
            .because("{ADR-NNN}: {business reason in one sentence}");

    // ... one block per ADR decision
}
```

## Rules for Rule Quality

- **Generic over specific:** `.haveSimpleNameEndingWith("Controller")` not `.haveFullyQualifiedName("com.example.order.controller.OrderController")`
- **One rule per concern:** do not combine unrelated constraints in one `.should()` chain
- **Actionable because():** the message must tell a developer what to do, not just what is wrong
- **Correct API:** use only ArchUnit 1.x API. `noClasses()`, `classes()`, `slices()`. Do not invent methods.
- **Cycle detection:** use `slices().matching("{root}.(*)..")` for bounded context cycle checks
- **Layer violations:** use `noClasses().that().resideInAPackage("..controller..").should().dependOnClassesThat().resideInAPackage("..repository..")`

## Critical API Constraints — Do Not Violate

**`onlyBeAccessedByClassesThat()` does not exist.** Use `onlyHaveDependentClassesThat()` instead.

```java
// WRONG — will not compile:
classes().that()...should().onlyBeAccessedByClassesThat().resideInAPackage("..controller..")

// CORRECT:
classes().that()...should().onlyHaveDependentClassesThat().resideInAPackage("..controller..")
```

---

**`.and()` after `dependOnClassesThat()` conditions is not valid.**

`dependOnClassesThat()` returns a `ClassesThat<ClassesShouldConjunction>`. Each condition method on it (`.haveSimpleNameEndingWith()`, `.resideInAPackage()`, etc.) returns `ClassesShouldConjunction` — which does NOT have `.and()` for chaining further target conditions.

**WRONG — will not compile:**
```java
.should().dependOnClassesThat()
    .haveSimpleNameEndingWith("Client")
    .and().resideInAPackage("..infrastructure..")  // ← INVALID
```

**CORRECT — one condition per `dependOnClassesThat()` call:**
```java
.should().dependOnClassesThat()
    .haveSimpleNameEndingWith("Client")
```

**CORRECT — use `.orShould()` for multiple independent target conditions:**
```java
.should().dependOnClassesThat().haveSimpleNameEndingWith("Client")
.orShould().dependOnClassesThat().resideInAPackage("..infrastructure..")
```

If you need to express "classes that match condition A AND condition B", write two separate rules — one per condition. Never chain `.and()` on the result of a `dependOnClassesThat()` condition.

## ArchUnit API Reference

Use this reference when writing rules. All APIs are from ArchUnit 1.x.

---

### Rule Entry Points

```java
import static com.tngtech.archunit.lang.syntax.ArchRuleDefinition.noClasses;
import static com.tngtech.archunit.lang.syntax.ArchRuleDefinition.classes;
import static com.tngtech.archunit.library.Architectures.layeredArchitecture;
import static com.tngtech.archunit.library.dependencies.SlicesRuleDefinition.slices;
```

| Entry point | When to use |
|---|---|
| `noClasses().that()...should()...` | "X must not depend on Y" — prohibition rules |
| `classes().that()...should()...` | "X must follow convention Y" — positive assertion |
| `layeredArchitecture()` | Define all layer rules in one declaration — **prefer this over individual noClasses() rules whenever ADRs define layers** |
| `slices().matching(...).should().beFreeOfCycles()` | Cycle detection across a set of slices |

---

### `layeredArchitecture()` — Full Layer Rule Declaration

**Always use this when an ADR defines a layered architecture.** It expresses all layer rules in one declaration and produces better error messages than individual `noClasses()` rules.

```java
// Status: ENFORCED CANDIDATE
@ArchTest
static final ArchRule FF_003_LAYER_RULES =
    layeredArchitecture()
        .consideringAllDependencies()
        .layer("Controller").definedBy("..controller..")
        .layer("Application").definedBy("..application..")
        .layer("Domain").definedBy("..domain..")
        .layer("Infrastructure").definedBy("..infrastructure..")
        .layer("Repository").definedBy("..repository..")

        .whereLayer("Controller").mayOnlyAccessLayers("Application")
        .whereLayer("Application").mayOnlyAccessLayers("Domain", "Repository")
        .whereLayer("Domain").mayNotAccessAnyLayer()
        .whereLayer("Infrastructure").mayOnlyAccessLayers("Domain", "Application")
        .whereLayer("Repository").mayOnlyAccessLayers("Domain")

        .because("ADR-003: One-way dependency rules must be maintained to keep each layer testable in isolation.");
```

Key methods on `LayeredArchitecture`:
- `.layer(name).definedBy(packagePattern...)` — declares a named layer
- `.whereLayer(name).mayOnlyAccessLayers(name...)` — restricts outbound dependencies
- `.whereLayer(name).mayNotAccessAnyLayer()` — no outbound dependencies (domain purity)
- `.consideringAllDependencies()` — also checks transitive dependencies (recommended)
- `.ignoreDependency(origin, target)` — temporarily exempt a known violation during migration
- `.as(description)` — override the rule description (optional)

### Cycle Detection with `slices()`

**The slice pattern must match the granularity you want to check:**

```java
// CORRECT — layer cycle detection WITHIN a service
// Slices: "controller", "application", "domain", "infrastructure", "repository"
@ArchTest
static final ArchRule FF_003_NO_LAYER_CYCLES =
    slices().matching("com.example.restaurant.order.(*)..")
        .should().beFreeOfCycles()
        .because("ADR-003: No cyclic dependencies between layers — cycles prevent isolated testing.");

// CORRECT — bounded context cycle detection ACROSS services
// Slices: "restaurant", "loyalty", "inventory"
@ArchTest
static final ArchRule FF_001_NO_CONTEXT_CYCLES =
    slices().matching("com.example.(*)..")
        .should().beFreeOfCycles()
        .because("ADR-001: Bounded contexts must not form dependency cycles.");
```

Rule of thumb: the `(*)` wildcard captures one path segment as the slice name.
- `com.example.restaurant.order.(*)..]` → slices are `controller`, `application`, `domain`, …
- `com.example.(*)..]` → slices are `restaurant`, `loyalty`, `inventory`, …

### `ClassesThat` Predicates (use after `.that()` or `.dependOnClassesThat()`)

```java
// Package-based
.resideInAPackage("..controller..")          // matches any segment named "controller"
.resideInAPackage("com.example.order.controller..") // full prefix match
.resideInAnyPackage("..controller..", "..application..")

// Name-based
.haveSimpleNameEndingWith("Controller")
.haveSimpleNameStartingWith("Abstract")
.haveSimpleNameContaining("Service")
.haveNameMatching(".*Repository")           // regex on fully-qualified name

// Type structure
.areAnnotatedWith(SomeAnnotation.class)
.areAnnotatedWith("com.example.SomeAnnotation")
.implement(SomeInterface.class)
.implement("com.example.SomeInterface")
.areAssignableTo(BaseClass.class)
.areInterfaces()
.areNotInterfaces()
.areEnums()

// Visibility
.arePublic()
.arePrivate()
.areProtected()
.arePackagePrivate()

// Combining predicates on the subject side (valid!)
.that()
    .resideInAPackage("..controller..")
    .and()                                  // .and() IS valid on ClassesThat after .that()
    .haveSimpleNameEndingWith("Controller")
```

---

### `ClassesShould` Methods (use after `.should()`)

```java
// Dependency rules
.dependOnClassesThat().<one predicate>       // depends on any class matching predicate
.onlyDependOnClassesThat().<one predicate>   // may ONLY depend on classes matching predicate
.notDependOnClassesThat().<one predicate>    // alias for noClasses().should()...

// Access rules (field access + method calls, stricter than "depend on")
.accessClassesThat().<one predicate>
.onlyAccessClassesThat().<one predicate>

// Naming / structure rules
.haveSimpleNameEndingWith("Impl")
.beAnnotatedWith(SomeAnnotation.class)
.implementInterface(SomeInterface.class)
.bePublic()
.notBePublic()

// Package rules
.resideInAPackage("..application..")
.onlyHaveDependentClassesThat().resideInAPackage("..controller..")
// NOTE: onlyBeAccessedByClassesThat() does NOT exist — use onlyHaveDependentClassesThat()
```

**`dependOnClassesThat()` vs `accessClassesThat()`:**
- `dependOnClassesThat()` — any compile-time reference (import, field type, method param, return type, extends/implements)
- `accessClassesThat()` — field reads/writes and method invocations only (subset of depends-on)
- Use `dependOnClassesThat()` for layer rules; use `accessClassesThat()` for stricter "no calls" rules

### Banned Library / Class Rules

```java
// Banned import (e.g. RestTemplate banned per ADR-017)
@ArchTest
static final ArchRule FF_017_NO_REST_TEMPLATE =
    noClasses()
        .should().dependOnClassesThat()
            .haveFullyQualifiedName("org.springframework.web.client.RestTemplate")
        .because("ADR-017: RestTemplate is banned. Use WebClient — it supports circuit breakers and reactive pipelines.");

// Banned package (e.g. direct JJWT usage outside platform-security-lib)
@ArchTest
static final ArchRule FF_SEC_NO_CUSTOM_JWT =
    noClasses()
        .that().resideOutsideOfPackage("com.example.platform.security..")
        .should().dependOnClassesThat().resideInAPackage("io.jsonwebtoken..")
        .because("All JWT validation must delegate to com.example.platform.security — custom JWT parsing is an unreviewed attack surface.");
```

### Cross-Context Import Rules

Generate one rule per context pair found in the codebase. The cycle check (`beFreeOfCycles`) only catches *bidirectional* dependencies — a one-way import from context A into context B's internals passes the cycle check. You must add an explicit prohibition rule for every such cross-context dependency.

```java
// ADR-001: bounded contexts must not import each other's internals.
// One rule per context pair — adapt package names to the actual contexts present.
@ArchTest
static final ArchRule FF_001_orders_must_not_directly_depend_on_loyalty =
    noClasses()
        .that().resideInAPackage("com.example.orders..")
        .should().dependOnClassesThat().resideInAPackage("com.example.loyalty..")
        .because("ADR-001: The orders context must not directly reference the loyalty context's internals. " +
                 "Cross-context communication must go through published APIs or async domain events.");
```

### Ignored / Suppressed Violations (migration window)

```java
// Temporarily ignore a specific violation while migration is in progress
@ArchTest
static final ArchRule FF_003_LAYER_RULES =
    layeredArchitecture()
        ...
        .ignoreDependency(
            com.example.restaurant.order.application.OrderService.class,
            com.example.restaurant.order.infrastructure.PaymentGatewayClient.class
        )
        .because("ADR-003: ...");
```

## What Not to Generate

- Rules that reference a specific class by fully qualified name (too narrow, breaks on rename) — use `haveSimpleNameEndingWith()` or package patterns instead
- Rules that cannot be expressed in ArchUnit (runtime behaviour, Kafka consumption, secret scanning — note these as "requires complementary tooling" in a comment)
- Rules for things the scan confirmed do not exist as violations (mark as ADVISORY, not ENFORCED CANDIDATE)
- `.and()` after a `dependOnClassesThat()` condition — see Critical API Constraints above
