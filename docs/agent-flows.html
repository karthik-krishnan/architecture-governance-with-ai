Runs every time
Input Source Code Java / TypeScript / Go / Ruby source files
↓
AI API Scanner Identifies endpoints, HTTP methods, path patterns, response types, annotations
↓
AI OpenAPI Generator Produces an OpenAPI 3.1 spec from the endpoint inventory
↓
Verify openapi-spec-validator Invalid spec sent back to model for correction ↺ up to 3× retry
↓
Output openapi.yaml → generated-specs/ Always regenerated — reflects current code state
Only when API Style Guide changes
Input Platform API Style Guide SHA-256 compared against hash embedded in the committed ruleset's first line
↓ unchanged → use cached ruleset, skip AI
AI Spectral Ruleset Generator Translates each style guide section into Spectral 6 YAML rules
↓
Verify spectral lint <probe spec> Checks ruleset is executable. On error, sends back to model — only fixes YAML syntax, never weakens rules. ↺ up to 3× retry
↓
Committed spectral-ruleset.yaml → inputs/ Governance artifact · SHA-256 of style guide in line 1 · never modified by the lint step

→
JUnit XML
→

Key insight: Lint is strictly read-only. The committed ruleset is a governance artifact — updated only when the style guide changes, never by the lint run.
